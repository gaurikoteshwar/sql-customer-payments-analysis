# SQL Customer Payments Analysis

A portfolio project demonstrating SQL and BigQuery skills through the analysis of customer, invoice, and payment data. This project includes query results, insights, and executive-level reporting artifacts.

## Project Overview

This project simulates a real-world business scenario where a company wants to understand its customers, invoices, and payments. The goal is to extract actionable insights, identify trends, and provide strategic recommendations.

**Key Objectives:**
- Analyze customer and invoice data to identify outstanding payments, top customers, and payment behaviors.
- Generate performance metrics and revenue trends over time.
- Provide executive-level insights and recommendations based on the analysis.

## Technologies Used

- SQL & BigQuery: for querying and analyzing structured data
- Looker: for reporting and visualization
- Canva: for presenting insights and executive summaries

## SQL Analysis

The SQL analysis was performed on customer, invoice, and payment data. All queries and results are documented in the PDF reports:

- **data_analysis_report_queries.pdf** – Screenshots of SQL queries executed  
- **data_analysis_report_insights.pdf** – Key insights and patterns observed  
- **executive_summary.pdf** – One-page executive summary  
- **presentation.pdf** – 5-slide summary presentation  

## Key Insights (Example)

- Identified top customers by outstanding balances and total payments.  
- Calculated average payment durations and on-time payment percentages.  
- Highlighted seasonal trends in payment delays.  
- Analyzed top-selling products/services and quarterly revenue patterns.
